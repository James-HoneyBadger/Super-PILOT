# Release Process

1. Ensure CHANGELOG.md is up to date.
2. Bump version in `VERSION` and any language-specific manifests if necessary.
3. Ensure CI is green on `main`.
4. Create a git tag: `vX.Y.Z`.
5. Draft GitHub release notes summarizing changes.
6. Publish release.

## Versioning

We follow Semantic Versioning (SemVer): MAJOR.MINOR.PATCH.
